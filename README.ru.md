# Ears Audio Toolkit — форк на Manifest V3

*[English version](README.md)*

Это неофициальный форк расширения **[Ears: Bass Boost, EQ Any Audio!](https://chrome.google.com/webstore/detail/ears-audio-toolkit/nfdfiepdkbnoanddpianalelglmfooik)** авторства Kevin King, переведённый с **Manifest V2** на **Manifest V3**, чтобы расширение продолжало работать после того, как Chrome/Chromium отключат поддержку MV2.

Миграция (манифест, service worker, offscreen document) сделана с помощью [Claude AI](https://claude.ai) (Anthropic).

> Это неофициальный форк силами сообщества. Он не связан с оригинальным автором и не одобрен им. Все заслуги за исходный дизайн, DSP и интерфейс принадлежат Kevin King.

## Что делает расширение

Ears позволяет в реальном времени эквализировать звук любой вкладки браузера: поднять бас, приглушить резкие высокие частоты, вывести вокал вперёд — и увидеть спектр частот того, что вы слушаете, в реальном времени. Используются биквадратные фильтры Web Audio API (полочные + пиковые фильтры), которыми управляют перетаскиваемые точки на интерактивном графике, плюс сохранение/загрузка пресетов и мгновенный Bass Boost в один клик.

## Установка (режим разработчика / unpacked)

Форк отсутствует в магазине расширений, поэтому он ставится как локальное распакованное расширение:

> **Уже установлено оригинальное расширение «Ears» из Chrome Web Store (или более старая копия этого форка)?** Сначала удалите его, и только потом устанавливайте эту версию. Если поставить новую копию поверх старой или просто нажать «Reload» после замены файлов на диске, браузер может продолжить работать со старым кодом - в частности, изменения в правах доступа и в фоновом скрипте так подхватываются не всегда. Чистая установка полностью исключает эту проблему:
> 1. Зайдите на страницу расширений (см. ниже) и полностью **удалите** старую версию Ears.
> 2. После этого выполните шаги ниже, чтобы заново загрузить эту версию через **Загрузить распакованное расширение**.

1. **Скачайте этот репозиторий** - кнопка *Code → Download ZIP* на GitHub, затем распакуйте архив.
2. Откройте страницу расширений браузера:
   - Chrome: `chrome://extensions`
   - Edge: `edge://extensions`
   - (у любого другого Chromium-браузера есть аналогичная страница `*://extensions`)
3. Включите **режим разработчика** (переключатель в правом верхнем углу).
4. Нажмите **Загрузить распакованное расширение** и выберите папку, где лежит `manifest.json`.

### Обновление после изменения/подтягивания нового кода

Простого нажатия **Reload** на карточке расширения не всегда достаточно - некоторые браузеры не всегда подхватывают новые permissions из `manifest.json` таким способом. Если что-то не работает или в консоли всплывает ошибка, связанная с правами:

1. Нажмите **Удалить** на карточке расширения (не просто Reload).
2. Заново нажмите **Загрузить распакованное расширение** и выберите ту же папку.

### Как пользоваться

1. Откройте вкладку, звук которой хотите эквализировать, нажмите иконку Ears и кликните **EQ Current Tab**.
2. Перетаскивайте точки на графике, чтобы менять форму эквалайзера; shift + перетаскивание точки вверх/вниз меняет ширину фильтра (Q).
3. Слева - ползунок общей громкости.
4. Сохраняйте понравившиеся настройки как именованный пресет или используйте готовый **Bass Boost** в один клик.
5. Переключайте **Spectrum Visualizer**, чтобы видеть частотный состав звука в реальном времени.

<details>
<summary><strong>Зачем нужен этот форк (нажмите, чтобы развернуть)</strong></summary>

Chrome (и большинство Chromium-браузеров) постепенно отключают расширения на Manifest V2. Оригинальный код Ears использовал постоянно живущую background-страницу MV2 с живым `AudioContext` внутри - это нельзя просто перенести в MV3: у service worker нет DOM, а значит нет ни `AudioContext`, ни `localStorage`. Этот форк перестраивает расширение вокруг обязательных примитивов MV3:

- **`manifest.json`** — `manifest_version: 3`, `action` вместо `browser_action`, service worker вместо постоянной background-страницы, и CSP, соответствующий требованиям MV3.
- **`bg.js`** (service worker) — хранит всё персистентное состояние (значения EQ-фильтров, гейн, пресеты, список активных вкладок через `chrome.storage`), опрашивает вкладки, работает с `chrome.tabCapture` и синхронизацией полноэкранного режима. У него нет доступа к DOM, поэтому всё, что связано со звуком, он делегирует offscreen-документу.
- **`offscreen.js`** / **`offscreen.html`** — скрытый offscreen-документ, который держит настоящий `AudioContext`, цепочку из 11 биквадратных фильтров и анализатор спектра. Он общается с service worker'ом исключительно через `chrome.runtime.Port`, поскольку offscreen-документы не имеют доступа к `chrome.tabs`, `chrome.storage` или `chrome.tabCapture` напрямую.
- **`popup.js`** / **`popup.html`** — почти не изменились: попап и раньше общался с фоновым скриптом только через `chrome.runtime.sendMessage`, поэтому ему не важно, что "фон" теперь разделён на два контекста.

#### Основные отличия от исходной MV2-версии

- Убрана старая аналитика Google (Universal Analytics, `UA-64913318-2`). Она и так уже не работала - Google отключила обработку данных Universal Analytics в июле 2023 года - а CSP страниц расширения в MV3 в любом случае больше не разрешает загружать удалённые скрипты.
- Пресеты хранятся одним объектом под одним ключом `chrome.storage.sync`, а не по отдельному ключу синхронизации на каждый пресет. Так проще, но при очень большой личной коллекции пресетов можно упереться в лимит 8 КБ на один элемент синхронизируемого хранилища.
- «Export Presets» теперь использует `chrome.downloads.download` вместо клика по `<a download>` в DOM (у service worker нет DOM, чтобы создать такую ссылку).
- Цепочка EQ-фильтров всегда полностью подключена последовательно (даже при 0 дБ), а не динамически подключается/отключается в графе. На звук это никак не влияет - фильтр на 0 дБ ничего не делает - просто немного больше нагрузки на процессор.
- Поскольку `AudioContext`/`getUserMedia` теперь живут в offscreen-документе (с жизненным циклом, независимым от service worker'а), список захваченных вкладок восстанавливается из `chrome.storage.local` и очищается при старте браузера, так как реальный поток захвата аудио в любом случае не переживает полный перезапуск браузера.

</details>

## Структура репозитория

```
manifest.json     Манифест MV3
bg.js             Service worker: хранилище, пресеты, вкладки, tabCapture, синхронизация полноэкранного режима
offscreen.html    Скрытый offscreen-документ (страница-хост)
offscreen.js      Аудио-движок: AudioContext, биквадратные фильтры, анализатор
popup.html        Разметка интерфейса попапа
popup.js          Логика попапа (отрисовка графика EQ, обработка перетаскивания, обмен сообщениями)
popup.css         Стили попапа (перенесено из оригинального репозитория — не менялось)
snap.svg-min.js   Встроенная библиотека Snap.svg для отрисовки графика EQ
ears*.png         Иконки панели инструментов/магазина (перенесено из оригинального репозитория — не менялось)
```
